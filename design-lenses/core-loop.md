# 透镜：核心循环（Core Loop）

状态：v0.1。项目锚点与个人化区间不在本文件——见各消费项目的 `Docs/design/design-lens-anchors.md`。

## 一句话本质

核心循环是玩家反复执行的"动作-反馈-决策"单元；这个透镜检查它的结构是否完好——有没有真决策、有没有断链、有没有被单一解统治——而不是检查它好不好玩。

## 何时取用（触发症状/任务）

- 设计稿写完了循环示意图，但说不清"玩家到底在按什么"。
- 玩法已经能跑，但测试者说"重复几次就腻了"，需要定位是循环本身缺内容还是曲线排布问题。
- 经济/资源系统上线后出现囤积、通胀，或某个数值一路涨没有回落。
- 怀疑存在"最优解"：老玩家全都在用同一套打法，其余选项形同虚设。
- 新增一层循环（例如从"单局"扩展到"局间成长"）之前，检查是否会稀释既有循环的决策含量。
- 复盘一个"逃逸"到验收环节才暴露的循环缺陷，回溯当初设计阶段这个透镜本该问出的问题。

## 它逼你回答的问题

### 1. MDA（机制-动态-体验）

MDA 把设计拆成三层：Mechanics（规则、数值、组件——设计师写的东西）、Dynamics（规则运行时产生的玩家行为模式）、Aesthetics（玩家实际的情绪体验）。设计师从 M 端写起，玩家从 A 端反向感知，中间的 D 层是两者唯一的翻译层，错位常在这里发生。

- 这条机制规则（M）在实际对局中会催生出什么行为模式（D）？这个行为模式是你设计时预想的那个吗？
- 玩家事后描述这个循环用的词（"爽""磨""憋屈"）对应的是哪条具体机制？如果说不出对应关系，说明 M→D→A 链条在你脑内也没打通。
- 是否存在一条机制，D 层行为完全符合预期，但产生的 A 层体验和你想要的南辕北辙（例如"平衡"的机制读起来像"无意义"）？
- 如果让一个没读过设计文档的人纯靠试玩复述规则，他复述出来的"隐规则"（D 层归纳）和你写的 M 层规则是否一致？
- 对回合制策略游戏：设计师是从"棋子的移动/攻击规则表"（M）想问题，还是从"这盘棋让人紧张"（A）想问题？如果只在 M 层反复调数值却说不出对应的 D 层行为改变了什么，说明改动是盲改。
- 对动作 Roguelite：某条 Buff/词条的机制文本（M）在实战中是否催生出玩家专门绕路刷这个词条的行为（D）？这种行为如果和"随机探索带来惊喜"的体验目标（A）相悖，说明词条设计已经跑偏。
- 团队内部对这个循环的"体验目标"（A）是否有一句共同的、可检验的描述？如果每个人心里的 A 层目标不一样，讨论 M 层数值改动时争的其实是各自脑内不同的 D 层假设，永远吵不出结果。

### 2. 动机循环（行动→反馈→奖励→投入）

这一层检查循环是否形成了完整的心理闭环：玩家做出行动，得到即时反馈，反馈兑现为奖励，奖励改变玩家对下一次投入的预期。断链发生在任一环节缺失或延迟过长。

- 从玩家按下这个动词到看见第一个反馈信号，中间隔了多久？这个延迟是否超过了玩家愿意"盲等"的阈值？
- 奖励发下来之后，玩家能不能在 3 秒内说出"这个奖励让我下一步该做什么不同的事"？如果奖励只是数字变大但不改变下一步决策，这是断链。
- 玩家的"投入"（时间/资源/心力）和他期待的"回报"之间，谁先承诺谁？如果投入必须先于任何反馈发生，循环第一次启动的门槛是否过高？
- 拿掉这个循环的数值包装（分数、特效、音效），玩家还愿不愿意为了纯粹的决策结果再来一轮？如果不愿意，说明循环靠的是反馈皮肤而不是真实动机。
- 反馈信号是否诚实？是否存在"看起来变强了但没有变强"的虚假正反馈，只是为了维持投入感？
- 玩家每个循环拍下的"动词"是什么？连续执行 10 次之后，这个动词还剩多少决策含量，还是已经退化成机械重复？
- 对回合制策略游戏：一次"结束回合"的确认动作之后，下一回合开局展示的信息是否足以让玩家修正上一回合的判断？如果每回合的信息增量趋近于零，投入（思考这一步）和回报（看到局面变化）之间已经脱钩。

### 3. 循环嵌套（micro/meso/macro）

同一份设计通常同时运行多层循环：秒级的操作循环（micro，例如一次攻击/一次落子）、局内的战术循环（meso，例如一场战斗/一局牌）、局间的成长循环（macro，例如养成/解锁）。这一节检查嵌套关系是否清晰，而不是互相稀释。

- 请把这个系统的 micro/meso/macro 三层循环各自用一句话写出来。写不出某一层，是这层真的不存在，还是被你忽略了？
- macro 层的奖励是否会反向改变 micro 层的操作决策含量？（例如养成把一次操作变成"数值碾压"，micro 循环名存实亡）
- meso 循环的一次完整周期时长，是否是 micro 循环时长的合理倍数（不是恰好 1 倍导致虚假分层，也不是几百倍导致玩家在 meso 尺度上感知不到 micro 的存在）？
- 玩家能否在只体验 micro+meso（不涉及 macro）的情况下，仍然觉得单局是完整的、值得玩的？如果单局本身空洞、全靠 macro 层的解锁诱因撑着，说明 meso 循环本身是空壳。
- 是否有某一层循环的胜负结果被另一层完全提前决定（例如战前配置就已经决定战斗结果），导致下层循环的执行沦为动画播放？
- 对动作 Roguelite：一次翻滚/一次攻击（micro）、一个房间的清怪节奏（meso）、一整轮 run 的 build 成型（macro）三者，玩家在描述"这局爽不爽"时说的是哪一层？如果玩家的评价永远只指向 macro（build 强不强），说明 micro/meso 两层可能已经沦为陪衬。
- 如果把 macro 层整体砍掉（不许养成、不许带任何跨局资源），meso 层的战术循环本身能不能撑起一次完整的、有峰有谷的单局体验？如果答案是否定的，说明当前的乐趣结构过度依赖 macro 层，meso 层是需要单独补强的短板。

### 4. 兴趣曲线（Interest Curve）

一局完整体验中的张力应当有峰有谷，而不是持平或单调递增/递减。这一节检查设计稿里是否明确规划过这条曲线，而不是任由随机数生成器决定节奏。

- 画出这个循环一局体验的张力曲线草图：开局、中段、高潮、收尾各处于什么强度？如果你画不出来，说明曲线是没设计过的，是系统自然涌现的副产品。
- 曲线的峰值出现在整局的什么位置？是否总是出现在同一个相对位置（例如永远是最后 10 秒），以至于玩家能预判"高潮点"从而提前放松警惕？
- 峰值之间是否存在明确的谷值（休息、喘息、低强度决策）？如果全程强度都很高，玩家的注意力资源是否被透支到无法感知峰值的存在？
- 曲线的形状是被设计规则强制的，还是完全依赖随机事件/对手行为？如果是后者，是否存在"曲线塌陷"的局（例如对手提前认输、随机没触发关键事件）导致整局平淡？
- 收尾阶段（无论输赢）是否给了玩家一个明确的、比中段更强的情绪落点？如果收尾比中段还平淡，玩家离场时记住的是什么？
- 对回合制策略游戏：一场对局里"局势明朗、结果已定但游戏还没结束"的回合数占比多少？这段"死局拖尾"越长，曲线在收尾前会被强行拉平成一条低强度直线。
- 对动作 Roguelite：单次 run 中，第一个精英/Boss 遭遇战出现的时间点是否随机到可能提前到玩家还没成型（macro 尚未展开）就撞上，导致这次曲线峰值"来得太早以至于没有铺垫"？

### 5. 经济学视角（source/sink、存量/流量）

把循环当作一个 Machinations 式的资源系统看：资源从哪个 source 产生，流向哪个 sink 消耗，中间的存量（stock）会不会无限堆积。这一节检查系统是否会自然收敛，还是必然走向囤积或崩溃。

- 列出这个循环里每一种核心资源的 source（产出点）和 sink（消耗点）。是否存在只有 source 没有对应 sink 的资源？这种资源长期看必然单调递增。
- 这个资源的产出速率和消耗速率，在正常游玩节奏下谁更快？如果产出恒定快于消耗，稳态就是囤积；囤积到后期是否让"获取"这个动作本身失去意义？
- 是否存在某个 sink 会因为前期囤积而在后期变得无关紧要（例如后期资源多到花不完，花钱决策不再是决策）？
- 资源转化链条中是否存在无损耗的循环兑换（A 换 B、B 又能换回 A 且不亏损）？如果存在，这条链会不会被拿来刷分或套利，绕开设计者原本想要的稀缺性？
- 如果把这个资源系统的产出速率整体上调 10 倍，游戏在多少局之后会失衡？这个数字如果很小（例如 3-5 局），说明系统对参数极度敏感，稳态是脆弱的。
- 对回合制策略游戏：行动点/资源点这类每回合刷新的流量资源，是否存在"攒起来不用，某回合一次性爆发"的策略空间？如果攒的收益永远大于即时花的收益，正常节奏的决策会被"憋大招"这一套外挂策略取代。
- 对动作 Roguelite：局内金币/材料这类 source，在一次 run 后期是否出现"能买的都买完了，剩下的钱没有 sink 可用"的情况？这段"钱花不出去"的时长占整局多少比例？

### 6. 退化策略狩猎（degenerate/dominant strategy hunting）

对循环里的每一个选择点做最优解分析：是否存在一个选项在几乎所有局面下期望收益都最高（统治性策略），或者反过来存在一个选项在几乎所有局面下都不会被理性玩家选择（死选项）。

- 把这个决策点的所有选项列出来。是否存在某个选项，在你能想到的任意局面组合下，换算期望收益都不低于其他选项？如果存在，这就是统治性策略。
- 反过来，是否存在某个选项，你需要构造非常刻意的局面才能让它成为最优解？这种"理论可行、实践永不发生"的选项就是死选项。
- 统治性策略一旦被高水平玩家发现并公开，其余选项的设计工作量（美术、数值、文案）是否就此打了水漂？这个风险你能接受吗？
- 这个循环里"随便选都差不多"的选择点占比多少？如果大部分选择点收益方差很小，说明决策含量整体偏低，循环在"提供选择的错觉"而不是"提供选择"。
- 死选项和统治性策略是否是同一枚硬币的两面（选了 A 必然导致 B 变成死选项）？修复统治性策略时，是否只是把问题转移给了另一个选项，而不是真正拉平了决策空间？
- 对回合制策略游戏：给定当前己方单位、资源、局面，理性玩家的"下一步最优行动"是否可以在不看对手意图的情况下被机械地算出来？如果是，说明这一步的决策实际上不依赖对局信息，是伪决策。
- 对动作 Roguelite：build 之间是否存在严格支配关系（某个 build 在所有关卡、所有节奏下都优于另一个 build，且没有明显的使用成本差异）？如果存在，其余 build 相当于陪跑内容。

## 红旗（越界即出"合理区间"）

- 存在统治性策略：某个选项在可枚举的绝大多数局面下期望收益都不低于其他选项。
- 存在死选项：某个选项在正常游玩中理性玩家从不选择，只有构造出刻意局面才会被选中。
- 奖励与投入断链：玩家获得的奖励不改变其下一循环可执行的决策集合（拿了等于没拿）。
- 反馈延迟超过玩家单次决策的耐心阈值，导致玩家在收到反馈前已经做出了下一次不相关的行动。
- 至少一种核心资源只有 source 没有对应量级的 sink，跑 N 局（N 为你在设计稿中设定的"一个成长周期"局数）后必然单调递增到失去决策意义。
- macro 层循环的产出可以让 micro 层的操作决策被数值直接碾压，使 micro 循环的执行退化为动画播放。
- 兴趣曲线在设计稿或数据回放中呈现单调直线（全程强度不变）而非有峰有谷。
- 存在无损耗的资源循环兑换路径（A→B→A 净收益为正或零亏损），可被用于套利而非用于设计者预期的资源分配。
- 拿掉数值/特效包装后，测试者明确表示不愿意再玩一次同样的决策结构。
- 单局收尾阶段的情绪强度低于该局中段，且没有设计意图支撑这个选择。
- 存在"憋大招"式的攒-爆发策略，其期望收益严格高于按节奏花费，且没有对应的时效性/风险成本抵消这个优势。
- M 层规则改动之后，设计者无法用一句话说出对应 D 层行为的预期变化——改动是在盲改而非有意图地调整体验。

## 局限（这个透镜看不见什么）

这个透镜只检查循环的**结构完好性**——有没有真决策、资源是否收敛、反馈是否兑现、曲线是否有意设计——它不能告诉你这个循环"好不好玩"。一个结构完全合格、无统治策略、无断链、曲线峰谷分明的循环仍然可能乏味，因为好玩涉及题材契合度、美术呈现、音效反馈手感、玩家个体偏好等这个透镜完全不覆盖的维度。结构合格是好玩的必要条件排查工具，不是充分条件保证书。

它也不负责判断循环之间的关系是否得当——例如"这个循环和整个游戏的题材/叙事是否契合"、"美术资源是否够撑起这个循环的视觉反馈"、"这个循环放进当前市场是否有商业竞争力"，这些问题需要其他透镜或专门的裁决流程回答，不要指望这一份透镜的产出能替代它们。最终"好玩与否"只能靠实际游玩裁决（playtest judgment），这个透镜的产出应当作为游玩裁决前的诊断清单，而不是替代游玩裁决本身。
